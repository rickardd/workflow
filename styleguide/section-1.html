1 #permalink KSS Documentation

"Markup:" will greate the code example block. It can be a referens to a file inside the sass folder or a commented example.

E.G: "Markup: froms/basic.html". Can use html or mustache .hbs files.

E.G: "Markup: google.com". Will print out the code.

If the word "Markup:" is not used the link above wont have the code markup but the link will appear like a link.

1.2 #permalink Modifiers

:hover - Subtle hover highlight. .stars-given - A highlight indicating you've already given a star. .stars-given:hover - Subtle hover highlight on top of stars-given styling. .disabled - Dims the button to indicate it cannot be used.

These ones will be generated inside the mustach variable {{modifier_class}}. One for each line

1.4 #permalink Issues

Known things that could create problems.

  1. Avoid comments starting with the word styleguide. That like asking for pars error.

  2. Blocknumbers over 99 can behave wierd. e.g styleguide 100.3 will be move this block under the item 1.

  3. If two group number are the same it can be a conflict. E.G "icon-colors 5.2" and "icon-sizes 5.2" can inharit each other properties.
